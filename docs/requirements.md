# Requirements

## Summary
Smart MEO Manager（SMM）は、Googleビジネスプロフィール運用を効率化するBtoB SaaSです。口コミ管理、AI返信案生成、人間承認フロー、MEO KPI可視化を一元提供し、多店舗運用を前提としたマルチテナント型ダッシュボードとして設計します。

## Target Users
- 飲食店運営企業
- 多店舗展開している事業者
- MEO対策を行っている企業
- Googleビジネスプロフィール運用を代行する事業者

## Problem
- 複数店舗の口コミ管理が分散し、対応漏れや返信遅延が起きやすい
- 返信品質やトーンが担当者ごとにブレる
- MEO指標を店舗横断で比較・分析しづらい
- 自動返信はブランド毀損リスクがあるため、人間確認前提の運用が必要

## Goals
- 口コミ管理を一元化する
- AIで返信作成工数を削減する
- 返信率改善とMEO改善に繋げる
- 多店舗の状況を比較可能にする
- 人間承認を前提に安全な運用基盤を作る

## Non-Goals
- 完全自動返信
- 汎用SNS統合をMVPで広げすぎること
- モバイルアプリをMVPに含めること
- 複雑なBIツール並みの分析機能を初期から入れること

## MVP Scope
- Clerk 認証
- Clerk Organizations によるマルチテナント
- ロール管理（ADMIN / MEMBER / READONLY）
- 店舗管理
- Googleビジネスプロフィール連携の土台
- 口コミ一覧、フィルタ、状態管理
- AI返信案生成
- 承認フロー
- 返信テンプレ管理
- KPIダッシュボード
- MEO分析の基本表示
- 新規口コミ通知、未返信アラート

## Monetization
- 店舗数ベースの月額課金
- 組織単位のプラン課金
- 将来的にAI生成数や高度分析機能で上位プラン化

## User Stories
- ADMINとして、組織配下の複数店舗口コミを一括管理したい
- MEMBERとして、未返信口コミに対してAI返信案を作り、承認依頼したい
- READONLYとして、口コミ状況やKPIを閲覧したい
- 運用担当として、星評価や返信状態で絞り込みたい
- 管理者として、店舗ごとの返信率や平均評価を比較したい

## Functional Requirements
- 組織・ユーザー・ロール管理
- 店舗登録・編集・Google Location ID 紐付け
- 口コミ一覧表示、検索、フィルタ、状態管理
- AI返信案生成と下書き保存
- 承認・却下・承認済み管理
- テンプレート管理
- KPI表示（総口コミ数、平均評価、返信率、未対応件数）
- グラフ表示（口コミ推移、検索・閲覧数）
- 新規口コミ通知、未返信アラート

## Non-Functional Requirements
- ページロード 3 秒以内
- ダッシュボード初期表示 2 秒以内
- APIレスポンス理想 500ms 以内
- 全APIで認証必須
- organizationId によるテナント分離
- Webhook署名検証必須
- 99.9%以上の可用性目標
- Vercel 前提の運用

## Risks
- GoogleビジネスプロフィールAPI制約
- マルチテナント分離ミスによる情報漏洩
- AI返信品質のばらつき
- Webhookや同期処理の不整合
- 多店舗データ増加によるパフォーマンス低下

## Success Metrics
- 口コミ返信率の改善
- 未対応口コミ件数の削減
- 店舗別の平均評価改善
- 口コミ対応にかかる工数削減
- 継続利用率と店舗追加数の増加
